    Stack Overflow is the largest online community for programmers to learn, share their knowledge, and advance their careers.

    As future data scientists, we cannot live without Stack Overflow when programming. To help us better understand and use this community, in this project, we are going to answer the following 3 questions:

    1. What can we know from comparing R questions with Python questions?
    2. How can we ask questions that are more likely to receive acceptable answers?
    3. Can we create a recommendation system to automatically generate tags for Stack Overflow questions?

    –––> scroll right

    In this project, we try to propose a feasible improvement on the currant classification engine to classify poodle dogs and fried chicken in terms of running cost (storage, memory and time) and prediction accuracy. The baseline model uses gradient boosting machine (gbm) with decision stumps on 5000 SIFT features. Our advanced model uses ensemble method (majority vote) of adaboost, random forest, knn, xgboost, and gbm on combination of 521 RGB features and original 5000 SIFT features, which enhances the accuracy rate by 21%.

    This is the last final project of Advanced Data Science which we are allowed to do whatever we want. Self-designed whole project flow, directed team of 5 to build an R Shiny app MelodySoup to generate chord progressions, designed logo. Conducted sequence analysis in R on chords of 990 songs using n-gram and tf-idf, visualized chord sequences by interactive chord diagrams and Sankey diagrams. Verified genre classifiablility by comparing prediction accuracy of Naïve Bayes and baseline model (improved by 107%) Introduced MelodySoup using ioSlides to class of 62 students, 2 professors, and 4 teaching assistants, scored highest marks in class.
